# S3.4 求积引擎与解析不变量测试

## 目的

实现一个与模型、probe 和 optimizer 解耦的求积引擎，并先在具有解析答案的低维问题上验证节点、权重、插值、逐坐标贡献、完备性和理论收敛阶。真实神经网络结果只有在这些不变量通过后才有解释意义。

## 前置条件

- G3-1 通过。
- Stage 1 提供稳定参数注册表和层/模块聚合映射。
- 本任务的解析测试可以在本机 CPU 开发；GPU 精度审计仍需 G3-0 的健康设备条件。

## 实施步骤

### 1. 实现求积规则注册表

1. 为每个规则分配稳定名称和版本。
2. 记录规则是单区间、复合等距还是 Gauss-Legendre。
3. 统一输出归一化到 \([0,1]\) 的节点和权重。
4. 记录子区间数、节点数和唯一梯度求值数。
5. 检查所有节点落在 \([0,1]\)。
6. 检查权重之和为 1。
7. 检查节点与权重长度一致。
8. 检查任何非有限节点或权重并立即失败。

### 2. 分别实现低成本规则

1. 实现左端点规则。
2. 实现右端点规则。
3. 实现中点规则。
4. 实现单区间梯形规则。
5. 实现单区间 Simpson 规则。
6. 为每种规则记录预期精确多项式阶和理论误差阶。
7. 验证单区间 Simpson 权重为标准 \(1:4:1\)。
8. 禁止把 Simpson 的三个节点误记成三个子区间。

### 3. 分别实现复合规则

1. 实现复合左端点。
2. 实现复合中点。
3. 实现复合梯形。
4. 实现复合 Simpson。
5. 验证复合规则的等距节点顺序。
6. 验证复合 Simpson 只接受正偶数子区间数。
7. 验证端点权重和内部奇偶节点权重。
8. 对非法子区间数提供明确错误，不自动向上取整。

### 4. 实现 Gauss-Legendre 规则

1. 使用受测数值库生成标准节点和权重。
2. 将节点从 \([-1,1]\) 映射到 \([0,1]\)。
3. 将权重按区间长度缩放。
4. 验证节点成对对称。
5. 验证 Q 点规则对次数不超过 \(2Q-1\) 的多项式积分精确。
6. 不手工硬编码高阶节点。
7. 记录数值库版本和 dtype。

### 5. 实现参数路径插值器

1. 只接收通过注册表校验的 pre/post 参数集合。
2. 逐张量计算总位移。
3. 对给定 \(\alpha\) 构造线性插值参数。
4. 验证 \(\alpha=0\) 恢复 pre-state。
5. 验证 \(\alpha=1\) 恢复 post-state。
6. 验证中点满足端点平均。
7. 保持参数名称、形状、dtype 和设备不变。
8. 检查共享参数不会被独立插值两次。
9. 对越界 \(\alpha\) 默认拒绝；反事实外推必须走单独接口和标识。

### 6. 实现节点梯度求值器

1. 接收固定参数状态和固定标量损失回调。
2. 在一次反向传播中获得全部研究参数梯度。
3. 对无梯度参数执行合同规定的处理。
4. 把分析梯度转换为规定的累加 dtype。
5. 记录节点 loss、梯度范数和非有限计数。
6. 节点结束后清理临时梯度和计算图。
7. 不为每个标量参数单独反向传播。
8. 不调用 optimizer 或修改数据游标。

### 7. 实现贡献累加器

1. 初始化与参数注册表同形状的路径平均梯度累加量。
2. 按节点权重逐张量流式累加梯度。
3. 使用更高精度累加路径平均梯度和全局贡献总和。
4. 计算 \(-\Delta\Theta\odot\bar g\) 的 signed 贡献。
5. 派生 positive 和 absolute 视图，但保留原始 signed 张量。
6. 计算参数张量、层和模块聚合。
7. 保存节点计数和唯一梯度求值计数。
8. 支持公共节点缓存，但按方法自身节点集合记账。
9. 缓存键必须包含端点、probe、参数路径、节点、精度和损失摘要。
10. 任何键字段不一致时拒绝复用缓存。

### 8. 构造常梯度解析测试

1. 构造线性标量损失。
2. 为多个坐标设置不同正负梯度。
3. 选择包含正、负和零位移的端点。
4. 解析计算逐坐标贡献。
5. 验证全部求积规则给出相同结果。
6. 验证贡献总和等于端点损失差。
7. 分别在 FP64 和 FP32 运行。

### 9. 构造二次与高阶多项式测试

1. 构造具有交叉 Hessian 项的多坐标二次损失。
2. 解析计算端点损失差和逐坐标路径贡献。
3. 验证梯形、Simpson 和适当 Gauss 规则对线性路径梯度精确。
4. 构造路径梯度为二次或三次多项式的损失。
5. 验证 Simpson 对相应次数的积分精确。
6. 构造超过低阶规则精确范围的四次/五次例子。
7. 验证复合规则随子区间增加收敛。
8. 验证 Gauss-Legendre 的精确次数边界。

### 10. 验证理论收敛阶

1. 为非精确多项式生成逐级加密网格。
2. 计算每级绝对积分误差。
3. 估计相邻加密的经验收敛斜率。
4. 验证左端点达到一阶趋势。
5. 验证中点和梯形达到二阶趋势。
6. 验证 Simpson 达到四阶趋势。
7. 在误差进入浮点平台期后停止用斜率作判断。
8. 分别记录 FP64 和 FP32 的平台期。

### 11. 构造抵消与边界测试

1. 构造总贡献接近零但单坐标贡献很大的正负抵消例子。
2. 验证完备性总和很小不会掩盖逐坐标错误。
3. 测试全零位移。
4. 测试部分参数不移动。
5. 测试端点损失变化接近零。
6. 测试参考贡献中大量零值和并列值。
7. 测试共享参数、冻结参数和无梯度参数。
8. 测试非法节点、非法权重和 Simpson 奇数子区间。
9. 测试 NaN/Inf loss、梯度和权重被明确拒绝。
10. 测试参数名称、形状或顺序不匹配被拒绝。

### 12. 验证幂等与无副作用

1. 对同一解析输入重复运行。
2. 比较节点序列、贡献和指标摘要。
3. 打乱规则注册顺序，验证结果不变。
4. 分别启用与禁用公共节点缓存，验证数值结果一致。
5. 在一个节点后模拟中断。
6. 恢复后验证最终结果与不中断运行一致。
7. 验证分析前后输入参数对象未被修改。

## 产出

- 版本化求积规则注册表；
- 参数线性插值器；
- 固定损失节点梯度求值器；
- 流式贡献与聚合累加器；
- 公共节点缓存和成本记账规则；
- 常梯度、二次、高阶多项式与抵消解析测试集；
- FP32/FP64 收敛阶和平台期报告；
- 异常输入、幂等、缓存和中断恢复测试。

## 核验标准

- 所有节点在合法区间、权重和为 1、规则元数据无歧义。
- 常梯度路径中全部方法逐坐标一致。
- 二次损失中梯形法完备性和逐坐标贡献达到相应 dtype 的数值精度。
- Simpson 和 Gauss-Legendre 通过各自多项式精确阶测试。
- 复合方法在浮点平台期之前表现出预期收敛阶。
- 正负抵消例证明测试会检查逐坐标误差，而非只检查总和。
- 非有限值、非法 Simpson 子区间、参数映射漂移和缓存身份冲突均明确失败。
- 缓存开关、中断恢复和重复运行不改变最终结果。

## Gate 与后续依赖

- 本子任务通过形成 **G3-3 求积实现 gate**。
- G3-3 与 G3-2 都通过后才能在真实模型上建立高精度参考。
- 理论阶测试只对解析光滑函数作硬 gate；真实网络路径进入浮点平台或非光滑区域时以经验收敛和交叉参考为准。

## 失败与恢复

- 若某规则权重或精确阶失败，修复后使所有受影响缓存失效并重跑解析测试。
- 若 FP32 平台期过高，先检查累加 dtype 和插值构造，再决定是否把特定参考状态升级到 FP64。
- 若公共节点缓存引入差异，禁用缓存完成正确性 gate，缓存优化作为独立修复，不影响数学实现。
