# S3.5 高精度参考、收敛与精度底噪

## 目的

为每个真实端点和固定 probe 建立一个经过节点加密、跨方法交叉确认且具有误差上界代理的高精度参考贡献。这个子任务避免重演旧归档中“固定 Gauss-16 直接作为零误差真值”的做法。

## 前置条件

- G3-2 与 G3-3 通过。
- 端点、probe、参数注册表和损失合同均已冻结。
- 真实模型节点求值在健康单卡 FP32 下可确定重现。
- 参考产物使用服务器大盘，不写入系统根盘。

## 实施步骤

### 1. 测定解析数值底噪

1. 汇总 S3.4 的 FP64 解析测试残差。
2. 汇总 S3.4 的 FP32 解析测试残差。
3. 分别记录插值误差、梯度误差、逐坐标贡献误差和贡献总和误差。
4. 记录大规模正负抵消对总和精度的影响。
5. 定义 FP64 和 FP32 的绝对底噪。
6. 定义按端点损失和贡献 \(L_1\) 质量缩放的相对底噪。
7. 数值底噪只从解析或重复确定性实验取得，不从候选方法误差反推。
8. 将底噪、稳定常数和硬件/库版本写入精度报告。

### 2. 定义两套独立参考家族

1. 选择 Gauss-Legendre 作为第一参考家族。
2. 为该家族设置逐级节点序列，例如 Q=8、16、32、64。
3. 选择复合 Simpson 作为第二参考家族。
4. 为该家族设置逐级偶数子区间序列，例如 n=16、32、64。
5. 明确每个 n 对应的唯一节点数。
6. 两个家族使用同一端点、同一 probe、同一精度和同一损失实现。
7. 不允许两个家族复用经过不同预处理的 probe。
8. 不把其中任一固定低阶结果预先声明为真值。

### 3. 逐级执行家族内加密

1. 先运行每个家族的最低参考级别。
2. 保存逐节点 loss、梯度范数和贡献聚合诊断。
3. 运行下一加密级别。
4. 计算相邻级别的归一化 \(L_1\)、\(L_2\) 和 \(L_\infty\) 差异。
5. 计算 active 参数 Spearman 和符号一致率。
6. 计算预注册 top-q overlap。
7. 计算层/模块质量分布差异。
8. 计算每一级别的完备性残差。
9. 只在相邻级别未收敛时继续加密。
10. 记录每次加密的新增梯度求值数和运行时间。

### 4. 进行跨家族交叉确认

1. 比较两家族各自最后两个已完成级别。
2. 比较最细结果的全参数向量误差。
3. 比较 active 参数排序和 top-q 集合。
4. 比较层和模块聚合。
5. 比较两者对固定 probe 端点损失差的完备性。
6. 检查是否存在“总和都准确但逐坐标分配不同”。
7. 跨家族不一致时，不选择残差较好的一方静默放行。
8. 先提高节点上限或排查损失/状态/精度问题。

### 5. 定义参考收敛误差预算

1. 把最后两级 Gauss 结果差定义为一个误差代理。
2. 把最后两级复合 Simpson 结果差定义为第二误差代理。
3. 把两家族最细结果差定义为交叉误差代理。
4. 取三者中的最大值作为保守参考不确定度。
5. 要求归一化 \(L_1\) 参考不确定度不超过候选方法允许误差的十分之一。
6. 在当前阈值模板下，该上限原则上不高于 0.1%。
7. 要求 active 参数 Spearman 原则上不低于 0.999。
8. 要求 top-1% overlap 原则上不低于 0.995。
9. 要求层/模块分布差异不超过候选层级预算的十分之一。
10. 要求完备性残差不超过由解析底噪导出的预算。
11. 将精确阈值在 pilot 后冻结，正式结果产生后不得放宽。

### 6. 选择规范参考表示

1. 在 pilot 配置中预先指定规范参考家族和最高级别。
2. 只有该家族内部收敛且另一家族交叉确认时，规范参考才有效。
3. 保存规范参考的原始 signed 参数贡献。
4. 保存 positive 和 absolute 派生视图。
5. 保存张量、层和模块聚合。
6. 保存参考不确定度和完备性残差。
7. 不把参考自身的 metrics 行写成零误差后用于美化汇总。
8. 报告中显式展示参考加密差和跨家族差。

### 7. 执行 FP64 真实模型审计

1. 从 pilot 的 early、middle、late 各预选至少一个 14M 端点。
2. 在看候选方法结果前固定这些端点。
3. 构造 FP64 分析模型副本。
4. 验证 FP64 端点损失和梯度有限。
5. 用选定参考级别运行 FP64 求积。
6. 比较 FP32 与 FP64 的端点损失、路径方向导数和贡献向量。
7. 比较 FP32 与 FP64 的完备性平台期。
8. 判断 FP32 误差是否显著低于候选方法预算。
9. 若 FP32 底噪过高，先改进累加与节点构造；不直接把 FP64 全量成本转嫁到正式实验。

### 8. 诊断未收敛路径

1. 检查所有节点是否使用同一 probe 和模型模式。
2. 检查节点参数是否精确落在线性路径。
3. 检查 loss reduction 和有效 token 权重。
4. 检查 dropout、buffer 和 RNG 是否固定。
5. 检查节点 loss 和方向导数沿 \(\alpha\) 是否出现尖峰或非有限值。
6. 检查贡献总和的正负抵消规模。
7. 检查 FP32/FP64 差异。
8. 在预算内提高节点上限并重新测试。
9. 仍不收敛时将该状态标记为 `reference_unresolved`。
10. 未收敛状态不得作为候选方法的误差分母或被静默排除。

### 9. 建立参考缓存与复用边界

1. 按端点、路径类型、probe、精度、节点和损失摘要建立缓存键。
2. 只复用完全匹配的节点梯度。
3. 记录缓存来源 run 和 SHA-256。
4. 参考配置改变时让相关缓存全部失效。
5. 候选方法可读取共同节点，但其成本报告仍按自身需要的节点数计算。
6. 正式重放抽查时绕过缓存，验证参考可重新生成。

## 产出

- FP32/FP64 数值底噪报告；
- Gauss-Legendre 与复合 Simpson 加密配置；
- 逐状态参考收敛表；
- 跨家族向量、排序、top-q、层级和完备性对照；
- 规范参考贡献及其不确定度；
- 选定状态的 FP64 真实模型审计；
- 未收敛路径诊断记录；
- 参考缓存 manifest 和无缓存重放证据。

## 核验标准

- 每个有效参考都经过至少两个相邻加密级别和一个不同家族交叉确认。
- 参考不确定度低于候选容许误差的十分之一。
- 参考自身的完备性残差落入由解析测试确定的预算。
- FP32 真实模型误差底噪显著低于候选 gate；否则有明确的精度升级方案。
- 报告同时展示参考误差代理，不把规范参考写成无条件真值。
- 未收敛状态有明确原因和状态码，不进入候选通过率分母。
- 无缓存抽查能重新生成相同参考。

## Gate 与后续依赖

- 本子任务通过形成 **G3-4 参考积分 gate**。
- 只有 G3-4 通过的 `path_state_id × path_type × probe_id` 才能评价低成本候选。
- 任一正式主状态参考未收敛都会阻断 G3-6 完整性 gate，除非它在预注册允许排除条件中且原因与方法结果无关。

## 失败与恢复

- 若两家族都不收敛，优先排查固定损失和状态污染，而不是无限增加节点。
- 若只有个别端点在预算内不收敛，保留失败证据并按预注册规则决定是否终止整个正式批次。
- 若 FP64 仍不改善，说明问题可能来自目标非确定或实现错误，返回 S3.2–S3.4 处理。
