# S3.2 数学对象、误差预算与指标合同

## 目的

在任何正式结果产生前，把路径、端点、固定损失、参数范围、求积方法、误差分解、排序语义和通过阈值写成不可随结果调整的合同。这个子任务用于消除 `general_plan.md` 中“一阶局部近似”和“左端点方法”可能被混用的歧义。

## 前置条件

- G3-0 通过。
- `docs/mathematics.md` 的现行修改已审查，路径积分符号和同批梯度平方公式正确。
- Stage 1/2 使用的损失归约、参数命名和主估计器目标已冻结。

## 实施步骤

### 1. 冻结端点与路径

1. 定义更新前参数为 \(\Theta_t\)。
2. 定义实际 optimizer step 后参数为 \(\Theta_{t+1}\)。
3. 定义总位移为 \(\Delta\Theta_t=\Theta_{t+1}-\Theta_t\)。
4. 固定主路径为 \(\Theta_t+\alpha\Delta\Theta_t\)。
5. 明确该路径是“实际更新端点之间的线性归因路径”，不是优化器内部连续轨迹。
6. 明确 full-update 路径可包含动量、预条件、裁剪和 weight decay 对端点的影响。
7. 明确 data-only 路径只有在位移分解经过独立不变量验证后才启用。
8. 明确任何反事实缩放路径都单独标记，不能混入主结果。
9. 明确实际位移不得再次乘学习率、裁剪因子或参数组缩放。

### 2. 冻结评价损失

1. 选择固定有限 probe 集损失作为主标量函数。
2. 固定序列长度、输入、labels、attention mask 和有效 token mask。
3. 固定损失是按序列平均还是按全 probe 有效 token 平均。
4. 若 probe 分 microbatch，固定基于有效样本或有效 token 数的加权合并公式。
5. 固定模型为 evaluation mode，关闭 dropout 等训练随机层。
6. 对不能关闭的随机性冻结随机状态，并让每个节点使用同一状态。
7. 固定是否包含正则项；主结果默认只评价数据损失。
8. 所有节点和两个端点必须调用同一损失实现和同一归约路径。
9. 端点损失差不得从训练日志中的另一个 batch 或另一个 reduction 读取。

### 3. 冻结参数范围

1. 生成参与研究的可学习参数清单。
2. 为每个参数记录规范名称、形状、dtype、所属层和模块。
3. 固定张量内展开顺序。
4. 检测共享底层存储和参数别名。
5. 规定共享参数只计一次，并记录其逻辑归属规则。
6. 将冻结参数与未参与优化的参数明确排除。
7. 将 buffer、optimizer state 和 scheduler state 排除出参数路径插值。
8. 为每个端点对指定唯一不可变的 `probe_buffer_snapshot`。
9. 要求全部路径节点以及两端 probe loss 都使用这份相同的 buffer 快照。
10. 断言 optimizer transition 前后的 buffer 相同；若某实现会改变影响前向的 buffer，则参数路径主合同失败，必须另建扩展状态路径合同，不能混入参数完备性等式。
11. 定义层总量、层每参数均值和层质量占比的聚合映射。
12. 为参数清单、buffer 快照和聚合映射计算稳定摘要。

### 4. 区分三个不同目标

1. 定义 `actual_path_left` 为实际端点路径的左端点求积。
2. 定义低节点和多节点方法都近似同一个固定 probe 路径积分目标。
3. 定义 `local_gradient_space_importance_u` 为 \(\eta_t s_t\mu_t^{\odot2}\) 的局部目标。
4. 明确局部 U 只在桥接分析中与路径分数比较。
5. 禁止用局部 U 的完备性残差评价路径求积。
6. 禁止因二者高度相关就宣称数学对象相同。
7. 仅在无动量、无 weight decay 的 population SGD 特例中说明退化关系。

### 5. 定义三类误差

1. 将固定 probe 下候选求积与收敛参考的差定义为求积截断误差。
2. 将不同独立 probe 之间的参考贡献变化定义为 probe 抽样波动。
3. 将解析测试、FP32/FP64 差异、累加和恢复误差定义为浮点/实现误差。
4. 规定增加节点只能用于判断求积误差，不能宣称减少 probe 抽样误差。
5. 规定增加 probe 只能用于判断抽样稳健性，不能替代参考加密。
6. 对同训练 batch 的评价只保留为同源诊断，不进入主 gate。
7. 定义参考不收敛、路径不确定和状态污染为结构性失败，不归入普通误差条。

### 6. 定义完备性指标

1. 计算贡献总和与固定 probe 端点损失下降的绝对残差。
2. 计算以端点损失下降绝对值加稳定常数为分母的相对残差。
3. 计算以参考贡献 \(L_1\) 质量为尺度的残差。
4. 当端点损失变化接近零时，禁止只报告普通相对残差。
5. 同时保存端点损失、贡献 signed sum 和 absolute mass。
6. 定义非有限损失、梯度或贡献为硬失败。

### 7. 定义逐参数向量指标

1. 计算归一化 \(L_1\) 误差。
2. 计算归一化 \(L_2\) 误差。
3. 计算按参考最大绝对值缩放的 \(L_\infty\) 误差。
4. 计算余弦相似度。
5. 根据数值底噪定义 active 坐标，再计算符号一致率。
6. 在 active 坐标上计算 Spearman，并用平均秩处理并列值。
7. 将未移动参数和参考近零参数单独计数，不让任意排序污染相关系数。

### 8. 定义排序与聚合指标

1. 分别报告 signed、positive 和 absolute 三种分数视图。
2. 规定 signed 是主数值对象，不把它归一化成概率质量。
3. 规定 positive 与 absolute 只用于操作性排序和非负质量分析。
4. 预注册多个 top-q 比例，而不是只选择一个方便的 top-k。
5. 对每个 top-q 计算 overlap 和 Jaccard。
6. 对层和模块分别计算总量、每参数均值和全模型质量占比。
7. 计算层/模块分布的总变差与 Spearman。
8. 不把层总量和层均值混成一个“层重要性”。

### 9. 定义成本指标

1. 把唯一梯度求值节点数作为算法成本主轴。
2. 单独记录前向次数、反向次数和端点损失求值次数。
3. 记录预热后的 wall-clock。
4. 记录峰值 GPU 显存。
5. 记录 CPU 内存、端点加载、状态恢复和结果写入时间。
6. 共享节点缓存只减少实验执行冗余，不改变单个候选方法的理论成本记账。

### 10. 冻结阈值策略

1. 解析测试先给出 FP32 和 FP64 数值底噪。
2. pilot 只能把底噪转换成更严格的正式阈值，不能查看正式结果后放宽。
3. 参考积分相邻加密差和跨方法差必须不超过候选容许向量误差的十分之一。
4. 候选方法的预注册上限原则上不得宽于：95 分位归一化 \(L_1\) 误差 1%。
5. 参数 active-set Spearman 的预注册下限原则上不得低于 0.99。
6. 各预注册 top-q overlap 的下限原则上不得低于 0.95。
7. 层级质量分布总变差的上限原则上不得高于 1%。
8. 另设最坏状态上限，防止中位数掩盖局部失败。
9. 若数值底噪本身无法支持上述上限，先修复精度或实现，不自动放宽研究标准。
10. 将最终阈值、稳定常数和 active-set 规则写入版本化配置并计算摘要。

## 产出

- Stage 3 数学合同；
- 固定 probe 损失与 token 加权合同；
- 参数清单、别名处理和层/模块映射 schema；
- full-update、data-only 与 local-U 的命名和解释边界；
- 完备性、向量、排序、聚合和成本指标字典；
- 三类误差分解说明；
- pilot 可填写但不可任意放宽的阈值模板；
- 配置摘要和变更审查规则。

## 核验标准

- 任一指标都能指向唯一数学目标、输入和归约方式。
- 同一 probe 的节点损失与端点损失来自同一个确定性标量函数。
- 两个端点和所有路径节点使用同一份已验收 buffer 快照；buffer 漂移不会被伪装成参数路径误差。
- actual-path 位移不再重复乘学习率或裁剪因子。
- AdamW full-update 路径和 local gradient-space U 的解释明确分开。
- 参数别名不会导致重复计数，参数顺序在保存和恢复后稳定。
- near-zero 端点损失变化有独立的尺度化残差。
- signed、positive、absolute 的用途和排序规则均已预注册。
- 阈值模板在正式结果产生前冻结，任何修改都产生新版本和原因记录。

## Gate 与后续依赖

- 本子任务通过形成 **G3-1 数学合同 gate**。
- G3-1 是端点/probe 管线和求积引擎实现的共同前置条件。
- 合同变更若改变目标损失、参数范围、主路径或 gate 阈值，必须废弃受影响的 pilot/formal 结果并使用新实验 ID 重跑。

## 失败与恢复

- 若 Stage 1/2 的损失归约或参数定义仍不明确，退回前置阶段处理。
- 若 full-update 与 data-only 的位移分解不能精确复现，只保留 full-update 主路径并把 data-only 标记为未启用。
- 若多个 ranking 视图给出不同结论，分别报告，不允许事后选择最有利的一种作为唯一主指标。
